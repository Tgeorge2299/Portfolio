My Homelab

My homelab is a single-node Proxmox environment used for learning, self-hosting, automation, and cybersecurity practice. It is designed to closely resemble real-world infrastructure while remaining flexible for experimentation.

Core Infrastructure

  • Hypervisor: Proxmox VE (single-node)
  • Primary Storage: ZFS pool tank (2 × 1TB HDDs)
  • Local Storage: 512GB NVMe for VMs and fast I/O workloads
  • Primary Firewall: pfSense appliance connected to Google Fiber

ZFS is used to provide data integrity, snapshots, and reliable shared storage across containers and virtual machines.

LXC Media Stack

Media services currently run in separate LXC containers for isolation, efficiency, and ease of management. All containers share a common ZFS mount managed by a dedicated Cockpit container.

  • Plex (media streaming)
  • qBittorrent (downloads)
  • Prowlarr (indexer management)
  • Radarr & Sonarr (media automation)
  • Overseerr (requests & discovery)
  • Cockpit (storage + mount management)
  • Homarr (homelab dashboard)

I plan to migrate this entire stack to Docker in the future once the architecture is finalized.

Virtual Machines

  • Ubuntu Server (general-purpose workloads)
  • Debian Server (Docker host)
  • Kali Linux (security testing)
  • Metasploitable (vulnerability practice)
  • Windows Server 2025
  • Windows 10
  • macOS VM
  • Ubuntu Workstation

Docker & Remote Access

A dedicated Debian VM runs Docker to host infrastructure services and web-facing tools.

  • Docker + Portainer
  • Nginx Proxy Manager
  • Cloudflare Tunnel (Zero Trust access)

Cloudflare Tunnel allows secure remote access to services without exposing ports or relying on traditional port forwarding.

Networking & Security Practice

An OPNsense VM is deployed on a separate internal network to practice VLAN configuration, firewall rules, and network segmentation without impacting production services.

  • VLANs & subnet segmentation
  • Firewall policy testing
  • Isolated attack / defense labs

Why I Built It This Way

This homelab prioritizes learning through real-world design choices: separation of concerns, infrastructure as services, secure access, and iterative improvement. It serves as both a production environment and a safe testing ground for new technologies.
